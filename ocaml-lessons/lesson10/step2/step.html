Module Types

You should have noticed by now that the top-level outputs the type of defined modules as it does for simpler values. The type of your Foo module was, unsurprisingly, the set of types of what you put in it, written between sig ... end keywords, with val to declare the name and type of included values inplace of let.

hint: to inspect a module from the top-level copy it into a new module; try for instance: module X = String.

As you can name custom types, you can also name custom module types, using this syntax: module type TypeName = sig ... end.

If you know C, you can think of OCaml modules as C compilation units and of OCaml module types as C header files (except of course that modules are much more powerful as we will see).

Now, as an exercise, let's try to define a module N1 which type is:

module type Number = sig
  val zero : int
  val add : int -> int -> int
end
